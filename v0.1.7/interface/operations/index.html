An interface for Lie group operations

LieGroups.AbstractGroupOperation ‚Äî Type
AbstractGroupOperation

Represent a type of group operation for a AbstractLieGroup $\mathcal G$, that is a smooth binary operation $‚àò : \mathcal G √ó \mathcal G ‚Üí \mathcal G$ on elements of a Lie group $\mathcal G$.

LieGroups.Identity ‚Äî Type
Identity{O<:AbstractGroupOperation}

Represent the group identity element $e ‚àà \mathcal G$ on an AbstractLieGroup $\mathcal G$ with AbstractGroupOperation of type O.

Similar to the philosophy that points are agnostic of their group at hand, the identity does not store the group $\mathcal G$ it belongs to. However it depends on the type of the AbstractGroupOperation used.

See also identity_element on how to obtain the corresponding AbstractManifoldPoint or array representation.

LieGroups.diff_inv! ‚Äî Method
diff_inv(G::LieGroup{ùîΩ, <:AbstractMultiplicationGroupOperation}, g, X)
diff_inv!(G::LieGroup{ùîΩ, <:AbstractMultiplicationGroupOperation}, Y, g, X)

Compute the value of differential $\mathrm{d}Œπ_{\mathcal G}(g)[X]$ of matrix inversion $Œπ_{\mathcal G}(g) := g^{-1}$ at $X ‚àà ùî§$ in the LieAlgebra $ùî§$ of the LieGroup G.

The (classical) differential $\mathrm{D}Œπ_{\mathcal G}(g): T_g\mathcal G ‚Üí T_{g^{-1}}\mathcal G$ reads

\[ \mathrm{D}Œπ_{\mathcal G}(g)[W] = -g^{-1}Wg^{-1} = -Xg^{-1} = -g^{-1}(gXg^{-1}) = -g^{-1}\mathrm{Ad}(g)[X] = V ‚àà T_{g^{-1}}\mathcal G,\]

see e.g. [Gil08]. To bring this back to the Lie algebra, we Write $V = g^{-1}Y ‚àà T_{g^{-1}}\mathcal G$ for some $Y ‚àà ùî§$ and obtain

\[ \mathrm{d} Œπ_{\mathcal G}(g)[X] = -\mathrm{Ad}(g)[X] ‚àà ùî§,\]

where we use $\mathrm{d}$ to denote the differential in the Lie algebra.

Abelian multiplication group operation

Since the commutativity of some Lie groups leads to a significant simplification of computations on those groups, the abelian multiplication group operation optimizes these cases. Additionally, it provides an interface for the abelian Lie groups. Some of these can are represented by isbits-types, which don't have mutating variants of the functions.
